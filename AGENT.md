# Agent Preferences

## 语言与推理规范（强制）

- **推理过程必须使用中文**：所有上下文分析、问题拆解、方案选型、架构思考等内部推理步骤，必须使用中文进行。这包括但不限于：
  - 代码变更前的分析与理解
  - 技术方案的比较与选择
  - 错误诊断与调试思路
  - 架构设计与重构规划
- **回答与说明默认使用中文**：向用户解释代码逻辑、汇报工作进展、提出建议等，一律使用中文。
- **代码注释使用中文**：新增或修改的代码注释应使用中文，与项目已有风格保持一致。
- **Commit 信息与文档可中英混用**：视具体上下文灵活选择，但技术术语保留英文原文。

> 此规则为最高优先级约束，任何情况下不得违反。
